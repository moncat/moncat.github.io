    # springboot怎么配置加密后的mysql用户名密码

    在 Spring Boot 中，你可以使用属性配置文件来配置加密后的 MySQL 用户名和密码。下面是一个示例：

    1. 首先，将加密后的用户名和密码添加到属性配置文件（如application.properties或application.yml）中：
    spring.datasource.url=jdbc:mysql://localhost:3306/mydb
spring.datasource.username=ENC(encrypted_username)
spring.datasource.password=ENC(encrypted_password)

    spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mydb
    username: ENC(encrypted_username)
    password: ENC(encrypted_password)

    注意，ENC(encrypted_username)和ENC(encrypted_password)是占位符，表示加密后的用户名和密码。

    1. 接下来，你需要配置一个自定义的属性解析器来解密这些属性。创建一个类，并实现 EnvironmentPostProcessor 接口，重写 postProcessEnvironment() 方法：
    import org.springframework.boot.SpringApplication;
import org.springframework.boot.env.EnvironmentPostProcessor;
import org.springframework.core.env.ConfigurableEnvironment;

public class CustomEnvironmentPostProcessor implements EnvironmentPostProcessor {
    @Override
    public void postProcessEnvironment(ConfigurableEnvironment environment, SpringApplication application) {
        // 解密加密的用户名和密码
        String encryptedUsername = environment.getProperty("spring.datasource.username");
        String decryptedUsername = decrypt(encryptedUsername);
        environment.getPropertySources().addFirst(new MapPropertySource("decryptedProperties",
                Collections.singletonMap("spring.datasource.username", decryptedUsername)));

        String encryptedPassword = environment.getProperty("spring.datasource.password");
        String decryptedPassword = decrypt(encryptedPassword);
        environment.getPropertySources().addFirst(new MapPropertySource("decryptedProperties",
                Collections.singletonMap("spring.datasource.password", decryptedPassword)));
    }

    在上述代码中，我们通过实现 EnvironmentPostProcessor 接口，并在 postProcessEnvironment() 方法中进行属性解密。你可以根据你的具体加密算法实现 decrypt() 方法。

    1. 最后，在应用程序的主类或任何其他配置类上添加 @AutoConfigureBefore 注解，以确保在应用程序启动过程中优先调用自定义的属性解析器：
    import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.boot.context.properties.ConfigurationPropertiesScan;

@SpringBootApplication
@AutoConfigureBefore(org.springframework.boot.context.config.ConfigFileApplicationListener.class)
public class YourApplication {
    public static void main(String[] args) {
        SpringApplication.run(YourApplication.class, args);
    }
}

    这样，当应用程序启动时，就会先调用自定义的属性解析器，将加密后的用户名和密码解密并配置到 Spring Boot 的环境中。
